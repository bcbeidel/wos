LLM agent planning has converged on three paradigms: reactive (ReAct), deliberative (Plan-and-Execute), and exploratory (Tree-of-Thought). Each makes fundamentally different trade-offs in how goals are decomposed, failures are handled, and progress persists across sessions. Choosing the right paradigm — or combining them — is the central architectural decision for any agent system.

## The Three Paradigms

**ReAct** interleaves reasoning and action in a tight loop: think, act, observe, repeat. There is no upfront plan. The agent decides its next action based on the most recent observation. This makes it highly adaptive to dynamic environments — it achieved a 34% absolute success rate gain on ALFWorld — but it has no lookahead. Decisions are locally optimal, and failure handling is purely in-loop correction.

**Plan-and-Execute** generates a full task list before executing any step. Plan-and-Solve prompting explicitly instructs the LLM to "first devise a plan, then carry out subtasks according to the plan." This produces ordered, checkable task lists. It consistently outperforms zero-shot chain-of-thought on math reasoning. The weakness: plans generated by LLMs can be wrong, incomplete, or based on hallucinated capabilities. Replanning is costly.

**Tree of Thoughts** explores multiple reasoning paths simultaneously using breadth-first or depth-first search. When a path fails, the agent backtracks to a branch point and tries an alternative. It achieved 74% on Game of 24 versus 4% for standard prompting. The cost is combinatorial: each decision point requires multiple LLM calls, making it impractical for long multi-step tasks.

## Goal Decomposition Strategies

Beyond the three paradigms, decomposition itself takes several forms. **Sequential decomposition** produces flat ordered task lists. **Hierarchical decomposition** (HuggingGPT, MetaGPT) maps top-level goals to domain-specific subtasks handled by specialist models or agents. **Adaptive decomposition** (Voyager) generates progressively harder subgoals based on current capabilities rather than decomposing everything upfront.

**DAG-based parallelism** (LLMCompiler) creates execution plans with explicit dependencies, dispatching independent tasks concurrently for up to 3.7x latency speedup over sequential approaches.

## Failure Recovery

Each paradigm implies a different failure strategy. ReAct corrects in-loop — the reasoning trace identifies issues and generates corrective actions. Plan-and-Execute replans from scratch when execution diverges. Tree of Thoughts backtracks to a prior branch point.

Two cross-cutting patterns improve recovery regardless of paradigm. **Verbal reflection** (Reflexion) stores textual analysis of failures in an episodic memory buffer, enabling the agent to avoid repeating mistakes across attempts — achieving 91% pass@1 on HumanEval. **SOP-based validation** (MetaGPT) catches cascading hallucinations by validating intermediate outputs against standardized procedures.

## Progress Tracking and Session Resumption

The academic literature undertheorizes artifact-based progress tracking, despite its practical dominance. Three artifact patterns emerge:

- **Skill libraries** (Voyager): completed tasks produce reusable code artifacts. Progress equals what the agent can do, measured by library contents. These persist across sessions without explicit checkpointing.
- **Reflection logs** (Reflexion): verbal self-critiques stored as text artifacts, enabling cross-attempt learning. Generalizes to cross-session learning when written to disk.
- **Plan documents with checkboxes**: common in production systems but absent from academic literature. The plan is both specification and state — an agent reads it to determine what is done and what remains.

Most academic approaches assume single-session execution. Artifact-based resumption — where the agent reads deliverable state rather than replaying conversation history — is more robust but less studied.

## Practical Implications

Production agent systems increasingly use hybrid approaches. The most effective pattern combines Plan-and-Execute for overall workflow structure with ReAct-style reasoning within each step. Key design decisions:

1. **Use plans for structured work, ReAct for exploration.** Well-defined multi-step tasks benefit from upfront planning. Information gathering and dynamic tasks benefit from reactive loops.
2. **Track progress through artifacts, not memory.** File system state, plan documents, and code artifacts are more durable and inspectable than serialized conversation history.
3. **Build approval gates at phase boundaries.** Human-in-the-loop patterns are well-established in engineering practice but lack formal treatment in the literature — implement them at artifact handoff points.
