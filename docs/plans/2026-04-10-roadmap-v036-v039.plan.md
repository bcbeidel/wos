# WOS Roadmap v0.36.0–v0.39.0

Ship the four-release roadmap defined in the 2026-04-10 design session.

## Goal

Deliver a self-improving WOS toolchain across four releases:

- **v0.36.0** — Wiki foundation: skill renames, wiki schema infrastructure, context file migration, universal ingest
- **v0.37.0** — Skill refresh: handoff contracts on all skills, skills refreshed against v0.35.0 research
- **v0.38.0** — Chain + build/audit family: audit-chain repair loop, build-skill/audit-skill, build-rule/audit-rule, build-subagent/audit-subagent, build-command/audit-command/build-hook/audit-hook, deprecate retrospective
- **v0.39.0** — Orchestrator: `/wos:audit` project health check, `WORKFLOW.md` lifecycle guide

## Approach

Each task corresponds to one GitHub issue. To execute a task:

1. **Set up isolation** — follow the branch strategy annotated on each task:
   - **Worktree** (no file overlap with siblings): `git worktree add ../wos-<name> -b feat/<name>` then open that directory in a new session. Allows true parallel execution without conflicts.
   - **Sequential** (overlapping files): checkout a shared branch, complete tasks one after the other in the same session.
2. `/wos:write-plan` — reference the issue number and the implementation detail in the issue comments
3. Review and approve the child plan
4. `/wos:execute-plan @docs/plans/<child-plan>.plan.md`
5. `/wos:validate-work`
6. `/wos:finish-work` — opens PR, references the issue

Check the task off here with the merge commit SHA once the PR is merged.

**Parallelism rule:** Tasks annotated `[worktree]` touch non-overlapping files — run them simultaneously in separate worktrees. Tasks annotated `[sequential]` share files — run them in order on a shared branch.

## Notes

- Each task produces a child plan via `/wos:write-plan`. The child plan is the execution artifact; this plan tracks sequencing.
- Standard `/wos:write-plan` prompt: `"write a plan for issue #NNN, include the implementation detail from the issue comments. Overall roadmap is tracked at docs/plans/2026-04-10-roadmap-v036-v039.plan.md — update Task N checkbox when this work merges."`
- Run worktree commands from the repo root (`~/Documents/git/wos`), not from inside an existing worktree.
- When a worktree PR merges, clean it up: `git worktree remove ../wos-<name>`.
- `distill` is not deprecated in this roadmap — it stays until `ingest` (Task 4) is shipped and validated. Reassess at v0.37.0.
